COMP4611 Homework 2: Cache Probing

 Due    Nov. 11, 2013 (10:00)

Write a program to access memory in certain ways, measure the performance, and infer the size of the cache on a real computer.

Note that your program must not directly read the internal CPU information about its cache size. Instead it should only measure memory access performance to infer the cache size.

Both individual and group submissions are permitted. If you submit as a group, the group formation should be the same as that for the project. Write clearly on the first page of the homework submission the members' names and student IDs.

Tasks

  1. Write and run the program to access memory in certain configurations (e.g., different sizes of memory blocks to be accessed) and collect the performance data (e.g., average access time) for the configurations.
  2. Use a plotting tool (e.g., Excel, gnuplot, MatLab) to visualize the relation between the performance data (e.g., average access time) and the configurations.
  3. Write a report with the following contents:
    • Experimental environment, including the type and model of the CPU.
    • Plot of the experimental data.
    • Analysis of the results you get, and your estimation of the cache size of the CPU.
    • The program source code
  4. Submit to CASS (https://course.cs.ust.hk/cass). Submit to HW2 and the report file should be named "report.doc".
